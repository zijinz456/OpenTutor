## Top 3 Marketing Highlights This Week

These are the most compelling features to surface in social posts, demos, and community threads over the next 7 days:

### 1. Cognitive Load Auto-Tuner with Per-User Adaptation
`apps/api/services/cognitive_load_tuning.py` (136 lines) — a sliding-window weight adjuster that prevents signal saturation. Students who always write short messages gradually reduce the brevity signal's influence on their personal score; an unusual behavior change gets full weight. No ML, no training data, just a bounded sliding window per user. This was featured in the 2026-07-07 HN Show HN post. It's a sharp, concrete implementation story with a clear tradeoff discussion.

**Demo angle:** Show the weight-decay constants and explain how a baseline learner stops crying wolf.

### 2. LLM-Graded Coding Questions (7th Question Type)
The quiz engine supports 7 question types including a `code_execution` type that routes to `ModelTier.FRONTIER` regardless of message length and uses LLM-rubric-free grading (correctness, completeness, clarity). The r/LocalLLaMA post from 2026-07-07 shows this is resonating. The "short code submission still routes to the bigger model" insight is technically interesting and underscores thoughtful tier routing.

**Demo angle:** Submit a short binary search implementation and show the graded JSON verdict in real time.

### 3. 10+ LLM Provider Support with Circuit Breaker Fallback
Multi-provider router with circuit breaker, automatic fallback, and local-first defaults (Ollama, LM Studio) alongside frontier providers (OpenAI, Anthropic, DeepSeek, Gemini, Groq, vLLM, OpenRouter). The router lives in `apps/api/services/llm/` and is genuinely unique among open-source learning tools — most require cloud API keys. This is the strongest first-impression hook for the privacy-conscious / self-hosting audience.

**Demo angle:** Show the provider selector in settings and demonstrate switching between Ollama and OpenAI with the same course.
